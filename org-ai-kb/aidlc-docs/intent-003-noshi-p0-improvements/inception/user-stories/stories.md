# User Stories — noshi P0 improvements（intent-003）

intent-001 の S-1..14 を保持しつつ、P0 改善のストーリーを追加。各ストーリーに AC（pass/fail）と Requirements トレース。
横断 AC（既存維持）: 本人スコープ（A01）、入力検証（A03）、汎用エラー、監査（A09）、モバイルファースト/WCAG AA。

## P0-1 / P0-3 期限ダッシュボード

### S3-1 お返し期限と残日数を知る
As a 個人の贈答管理者, I want もらった贈答ごとにお返しの期限と残日数を見たい, so that 出し忘れを防げる。
- AC1: 受領レコードに用途別の標準期限が算出される（香典+49日、出産/結婚/快気/一般慶事+1ヶ月、お中元・お歳暮は期限なし）。
- AC2: 各お返し対象に残日数（例: のこり5日／期限超過）が表示される。
- AC3: 期限なし（中元・歳暮）はお返し対象に出さない。
- Requirements: FR-3-1

### S3-2 ホームで期限順に把握する
As a 個人の贈答管理者, I want ホームでお返しを期限の近い順に見たい, so that 何を急ぐべきか一目で分かる。
- AC1: ホームの主役がお返し期限の一覧（残日数順カード）である。
- AC2: 収支・差分の表示が無い。
- AC3: 各カードは 相手・用途・金額・残日数・状態 を表示し、タップでお返しフローへ。
- Requirements: FR-3-2

## P0-2 撮影レビュー

### S3-3 要所だけ確認して保存する
As a 個人の贈答管理者, I want AIが自信のある項目はそのまま、怪しい項目だけ確認したい, so that 手早く正確に記録できる。
- AC1: 高信頼の項目は確定済みとして静かに表示される。
- AC2: 低信頼（要確認）の項目だけが強調される。
- AC3: 要確認が無ければ全項目レビューなしで保存できる。
- Requirements: FR-3-3

## 負債#1 given 除外

### S3-4 あげた贈答はお返し対象に出ない
As a 個人の贈答管理者, I want 自分が「あげた」贈答は未完了お返しに出したくない, so that お返しが必要なものだけに集中できる。
- AC1: direction=given のレコードは未完了お返し（pending）に現れない。
- AC2: given は台帳には従来どおり記録・表示される。
- Requirements: FR-3-4

## system story

### S3-5 期限計算サービス
As the noshi サービス, when 受領レコードが作られたとき, it must 用途別の標準期限を受領日起点で算出し、残日数を返す。
- AC1: 期限は確定的に算出される（日本時間・日付ベース）。occurred_at 未設定時は作成日でフォールバック。
- AC2: お返し不要の用途は期限を持たない。
- Requirements: FR-3-1, FR-3-2

## カバレッジ補足
- セキュリティ/abuse（S-10..13）・プライバシー（S-14）は intent-001/002 のものを維持（本 intent で変更なし・再記述しない）。
- FR-3-1.4（期限上書き）は将来拡張として保持し、本 intent のストーリーでは残日数表示・並びを優先（非ストーリー化の明記）。
