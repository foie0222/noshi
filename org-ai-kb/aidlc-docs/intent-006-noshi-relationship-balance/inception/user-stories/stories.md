# User Stories — noshi relationship balance（intent-006）

intent-001..005 を保持。N1 を追加。各 AC（pass/fail）と Requirements トレース。横断 AC: 本人スコープ・AA。

## N1 おつきあいバランス

### S6-1 相手別のやりとりを把握する
As a 個人の贈答管理者, I want 相手ごとに もらった/あげた/差分と最後のやりとり時期を見たい, so that おつきあいの状況を把握できる。
- AC1: 相手別に received合計/given合計/差分/最終やりとり日が表示される。
- AC2: 本人データのみ（A01）。
- Requirements: FR-6-1

### S6-2 偏りにそっと気づく
As a 個人の贈答管理者, I want もらってばかりで長くお贈りしていない関係に気づきたい, so that 失礼なく関係を続けられる。
- AC1: 偏り状態（均衡/もらい超過/あげ超過）が分かる。
- AC2: もらい超過 かつ 最終やりとりから一定期間（既定180日）の関係が「気になる関係」として控えめに示される。
- AC3: 損得ではなく「関係のメンテナンス」のやさしい言い回し（責めない）。
- Requirements: FR-6-2

### S6-3 おつきあい一覧
As a 個人の贈答管理者, I want マイページでおつきあいの一覧を見たい, so that 一目で気になる関係に気づける。
- AC1: マイページに相手別バランス一覧があり、気になる関係が上位に控えめ表示される。
- AC2: 各行から相手の記録へ辿れる（または相手名で台帳を絞れる）。
- Requirements: FR-6-3

## system story
### S6-4 バランス集計
As the noshi サービス, when 台帳を評価するとき, it must 相手別に received/given/差分/最終やりとり日を確定的に集計し、偏りを分類する。
- AC1: 集計・分類は確定的（日付・しきい値）。本人データのみ。
- Requirements: FR-6-1, FR-6-2

## カバレッジ補足
- security/abuse・プライバシーは維持・再記述しない。
