# User Stories — noshi P1 improvements（intent-004）

intent-001..003 を保持しつつ P1 を追加。各 AC（pass/fail）と Requirements トレース。
横断 AC（維持）: 本人スコープ(A01)・入力検証(A03)・監査(A09)・WCAG AA。

## P1-1 弔事/慶事トーン

### S4-1 弔事は静かなトーンで
As a 個人の贈答管理者, I want 香典など弔事の画面は落ち着いたトーンで表示してほしい, so that 悲しみの文脈で無神経に感じない。
- AC1: 用途が弔事（香典/御霊前/法事/弔慰）の文脈は配色が墨/グレー系に切り替わる。
- AC2: 弔事のコピーは控えめになる（祝い向けの明るい語を避ける）。
- AC3: 慶事は現状トーンを維持。コントラストは AA を満たす。
- Requirements: FR-4-1

## P1-2 信頼の可視化

### S4-2 自分だけが見えると分かる
As a 個人の贈答管理者, I want 相手の名前を入れるとき「自分だけが見られる」と分かりたい, so that 機微な情報を安心して預けられる。
- AC1: 氏名入力の近くに「あなただけが見られます🔒」相当の表示が出る。
- AC2: consent・設定にも本人スコープ・暗号化・削除権の安心文がある。
- AC3: 表示のみで実体の認可（本人スコープ）は不変。
- Requirements: FR-4-2

## P1-3 贈与税110万枠の気づき

### S4-3 今年の贈与税の枠を意識できる
As a 個人の贈答管理者, I want 今年もらった対象合計と110万円枠への接近を知りたい, so that 高額贈与の税の不安に早めに気づける。
- AC1: 暦年（1/1〜12/31）の「もらった」対象合計が表示される（香典・お中元・お歳暮は除外）。
- AC2: 「110万まであと◯円」、超過時は注意が表示される。
- AC3: 「税アドバイスではない・気づき」の免責が併記される。本人データのみ集計。
- Requirements: FR-4-3

## system story

### S4-4 弔事分類と年間集計
As the noshi サービス, when 用途や受領レコードを評価するとき, it must 用途を弔事/慶事に分類し、暦年の対象もらった合計（社会通念上の贈答を除く）を算出する。
- AC1: 分類・集計は確定的（キーワード・暦年・日本時間）。
- AC2: 集計は本人データのみ（A01）。
- Requirements: FR-4-1, FR-4-3
