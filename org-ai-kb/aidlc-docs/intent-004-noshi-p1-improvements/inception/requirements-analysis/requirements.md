# Requirements — noshi P1 improvements（intent-004）

## Intent summary
- Type: feature（brownfield 改善）/ Scope: 単一サービス noshi-service / Classification: brownfield / Affected repos: noshi
- 源泉: design-review 001 の P1。intent-001..003 の FR/NFR を保持しつつ改修。

## Functional requirements（新規・変更）

- **FR-4-1 弔事/慶事トーン（P1-1）**
  - FR-4-1.1 用途を弔事/慶事に分類する（弔事: 香典・御霊前・法事・弔慰 等）。
  - FR-4-1.2 弔事文脈の画面は配色を静かに（朱→墨/グレー系）切り替え、コピーを控えめにする。
  - FR-4-1.3 慶事は現状のトーンを維持。モーション切替は本 intent では行わない。

- **FR-4-2 信頼の可視化（P1-2）**
  - FR-4-2.1 相手（氏名）入力の近くに「あなただけが見られます🔒」相当の安心表示を出す。
  - FR-4-2.2 consent・設定にも同旨の安心文（本人スコープ・暗号化・削除権）を示す。
  - FR-4-2.3 表示のみで、実体の認可（本人スコープ A01）は不変。

- **FR-4-3 贈与税110万枠の気づき（P1-3）**
  - FR-4-3.1 暦年（1/1〜12/31）で「もらった」合計を集計する。**社会通念上の贈答（香典・お中元・お歳暮）は対象から除外**する。
  - FR-4-3.2 基礎控除110万円に対し「今年の対象合計 / 110万まであと◯円 / 超過時は注意」を**専用サマリ（控えめ）**で表示する。
  - FR-4-3.3 これは**税アドバイスではない・気づき**である旨の免責を明記する。本人データのみ集計（A01）。

## Non-functional requirements
- NFR は intent-001..003 を維持。本人スコープ（A01）・入力検証（A03）・汎用エラー・監査（A09）・分類不変。
- トーン分類・年間集計・枠判定は確定的（日本時間・暦年）。集計は本人のもらったデータのみ。
- アクセシビリティ: 弔事トーンでもコントラスト AA を満たす。

## Assumptions
- 技術スタック不変。通知/リマインドは引き続きスコープ外。
- 贈与税の判定は概算の気づきであり、正確な課税判断・申告要否は専門家に委ねる（免責）。
- 弔事の判定はキーワードベース（香典/御霊前/法事/弔慰 等）。

## Out of scope
- P2（IA再設計・水引モーション・オンボーディング・a11y全面・コピー全面）
- 正確な税額計算・申告支援、税率適用
- フルテーマ切替（モーション・アイコン・余白）
