# Requirements — noshi P0 improvements（intent-003）

## Intent summary
- Type: feature（brownfield 改善）/ Scope: 単一サービス noshi-service / Classification: brownfield（既存 backend・frontend を改修）/ Affected repos: noshi
- 源泉: design-review 001 の P0。既存の FR/NFR（intent-001 requirements.md）を保持しつつ改修。

## Functional requirements（新規・変更）

- **FR-3-1 お返し期限ルール（P0-1）**
  - FR-3-1.1 受領レコードに、用途別の標準期限を **受領日(occurred_at)起点**で算出する:
    香典=+49日（四十九日）、出産祝い/結婚祝い/快気祝い/一般慶事=+1ヶ月、お中元・お歳暮=期限なし（お返し不要）。
  - FR-3-1.2 各お返し対象に **残日数** を表示する（例: のこり5日 / 期限超過）。
  - FR-3-1.3 期限のある未完了お返しを **期限の近い順** に並べる。期限超過は強調する。
  - FR-3-1.4 標準期限はユーザーが上書きできる（将来拡張可、MVP は表示・並びを優先）。

- **FR-3-2 ホーム=お返し期限ダッシュボード（P0-1, P0-3）**
  - FR-3-2.1 ホームの主役を「お返し期限」の一覧（残日数順カード）にする。
  - FR-3-2.2 収支（もらった/あげた/**差分**）の表示を **撤去** する（贈答を損得で見せない）。
  - FR-3-2.3 各カードは 相手・用途・金額・残日数・状態 を表示し、タップでお返しフローへ。

- **FR-3-3 撮影レビューの「要所だけ確認」（P0-2）**
  - FR-3-3.1 抽出結果のうち **高信頼の項目は確定済み**として静かに表示する。
  - FR-3-3.2 **低信頼（要確認）の項目だけ**を強調し、確認を促す。
  - FR-3-3.3 全項目の強制レビューはやめる（要確認が無ければ即保存可能）。

- **FR-3-4 given（あげた）をお返し対象から除外（負債#1）**
  - FR-3-4.1 direction=given のレコードは「未完了お返し」イベントを作らない（received のみ pending 対象）。
  - FR-3-4.2 given は台帳には従来どおり記録・表示される。

## Non-functional requirements
- NFR は intent-001/002 を維持。本人スコープ（A01）・入力検証（A03）・汎用エラー・監査（A09）・restricted非ログ・外部送信最小化は不変。
- 期限計算は確定的（タイムゾーンは日本、日付ベース）。差分撤去でデータ分類・保存は不変。
- 性能・モバイルファースト・アクセシビリティ（WCAG AA）維持。

## Assumptions
- 技術スタック不変（Python/FastAPI + React/TS + DynamoDB、ローカルは InMemory + モック）。
- 通知/リマインドの push はスコープ外（**画面表示のみ**で期限を伝える）。
- 期限の基準日は occurred_at（受領日）。未設定時は作成日でフォールバック。

## Out of scope（本 intent では除外）
- まとめ撮り（バッチ・イベント単位の複数レコード）
- P1（弔事/慶事トーン、信頼の可視化、贈与税110万枠）、P2（IA再設計、水引モーション、オンボーディング、a11y強化、コピー全面見直し）
- push 通知・リマインド配信
