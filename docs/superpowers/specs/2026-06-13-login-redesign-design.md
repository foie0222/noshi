## 1. 背景・コンテキスト（なぜこれをやるのか）

noshi はサービスの説明を3つの層で持っている。

| 層 | 文言 | 置き場所 |
|---|---|---|
| サービス名 | noshi | ロゴ / favicon / `<title>` |
| 何をやるか（説明） | 贈りものの記録と、お返し選び | `<title>`：`noshi｜贈りものの記録と、お返し選び` |
| コンセプト（情緒） | 大切な人との縁を、長く美しく。 | `index.html` の meta description / og:description（#149 で設定） |

ところが**ログイン画面のタグラインだけ「贈答を、ちゃんと続けられる。」という旧方針（記録・継続寄り、やや事務的なトーン）のまま**になっている。

これが問題なのは、ログイン画面が「サービスに触れる最初の画面」だからである。ここで打ち出す言葉が、対外コピー（OGP で共有されたときに見える言葉）や、プロダクトが目指す世界観（縁・情緒）とズレていると、

- 初めて来た人が受け取るブランドの第一印象が、SNS で見た印象（OGP）と食い違う
- 「縁を大切にする」という情緒的な価値づけと、「記録を続ける」という機能的な言い方が混在し、何のためのサービスなのかが曖昧になる

要するに、**プロダクトの顔である画面が、確定済みの方針と違うことを言っている**状態の解消が目的である。

## 2. ゴール / ノンゴール

**ゴール**
- ログイン画面のヘッダを、上記3層（名前・説明・コンセプト）で構成し、タイトル・OGP・ログイン画面の3箇所が同じ言葉で揃う状態にする。
- 既存の「和・温かみ・上品」「年配の方にも読みやすい」というデザイン方針を保ったまま、見た目の品を一段上げる。

**ノンゴール（やらないことを明示）**
- ログインカード（Google / メール / LINE）の機能・レイアウト変更。今回は触らない。
- 配色・トーン・全体構成の全面刷新。
- `index.html` / OGP の文言変更（既に目標の文言なので変更不要、一致確認のみ）。
- 認証ロジック・コピー以外の文言（エラーメッセージ等）の変更。

## 3. 設計

### 3.1 ヘッダの構成

上から順に（変更は 3〜5）：

1. ロゴ `<Logo variant="full" size={40} />` — 現状維持
2. `N O S H I`（`.brand-en`）— 現状維持
3. **大切な人との縁を、長く美しく。**（コンセプト・明朝・主役）
4. 細い区切り線（淡いサンド色・幅 ~38px）
5. 贈りものの記録と、お返し選び（説明・小・ゴシック・サブ色）

旧 `.muted` の1行「贈答を、ちゃんと続けられる。」は廃止。

### 3.2 `frontend/src/App.tsx`（ログイン画面ヘッダ, 現 771–779 行付近）

変更前:
```tsx
<div className="brand"><Logo variant="full" size={40} /></div>
<div className="brand-en">N O S H I</div>
<p className="muted" style={{ textAlign: "center" }}>
  贈答を、ちゃんと続けられる。
</p>
```

変更後:
```tsx
<div className="brand"><Logo variant="full" size={40} /></div>
<div className="brand-en">N O S H I</div>
<p className="login-concept">大切な人との縁を、長く美しく。</p>
<div className="login-divider" aria-hidden="true" />
<p className="login-desc">贈りものの記録と、お返し選び</p>
```

### 3.3 `frontend/src/styles.css`（新クラス3つ。`.brand-en` 定義の直後に追加）

```css
.login-concept {
  font-family: var(--font-display);   /* Shippori Mincho B1 */
  font-weight: var(--fw-bold);         /* 700 */

  color: var(--text-body);
```

- 参照トークンは実在確認済み：`--font-display`/`--font-body`/`--fw-bold`/`--fs-h3`/`--fs-sm`/`--text-body`/`--text-sub`/`--border-default`/`--lh-relaxed`/`--ls-wide`/`--space-3`。

## 4. 検討した代替案と却下理由（Alternatives Considered）

ブレストで比較し、以下を採用／却下した。実装時に「なぜこの形か」を蒸し返さないために記録する。

### 4.1 リデザインの範囲
- **A. コピー刷新だけ** — 最小だが、せっかく方針を直すのに見た目のチグハグ（タグライン1行だけ差し替えた不自然さ）が残る。却下。
- **B. コピー＋ビジュアル底上げ（採用）** — コピーを直しつつ階層・余白・区切り線で品を整える。レイアウト構成は変えないので影響範囲が限定的。
- **C. 全面刷新** — 配色・トーンから作り替え。インパクトは大きいが、「年配の方にも読みやすい」「和フラット」という既存方針と衝突しやすく、作業量も過大。却下。

### 4.2 タグライン（コンセプト文）
- **A. 「大切な人との縁を、長く美しく。」（採用）** — OGP / meta と完全一致し、3層の一貫という目的に最も忠実。
- B. 「贈りものの縁を、末長く。」 — 短く明朝映えするが、OGP と別文言になり一貫性の目的から外れる。却下。
- C. 「いただいた心に、きちんとお返しを。」 — お返し機能を前面に出せるが説明的で、コンセプト（情緒）の役割としては機能寄り。却下。
- D. 「贈り、贈られ、つながっていく。」 — 温かいが、これも OGP と別文言。却下。

> 補足：採用理由の核心は「すでに対外的に出している文言（OGP）に画面を合わせる」こと。新しい名コピーを作ることが目的ではない。

### 4.3 ヘッダの情報配置
- A. コンセプト（大）→ 説明（小）。情緒先行。
- B. 説明（小）→ コンセプト（大）。読み下しは自然だが、説明が先に立つと情緒の主役性が弱まる。
- **C. コンセプト（大）→ 区切り線 → 説明（小）（採用）** — 情緒を主役に保ちつつ、区切り線で「情緒」と「実務的な説明」を視覚的に分け、和の上品さが出る。明朝＋細い罫線の相性が良い。

### 4.4 ビジュアルの強度
- **案1. 控えめ（採用）** — 既存の和フラット路線を磨く。グラデ・影なし。デザインシステムの方針に忠実。
- 案2. 演出あり（上部余白拡大・区切り線を朱赤に・カードに淡い影） — "作品感"は出るが、デザインシステムの「グラデ・テクスチャ・影なし」方針から外れる。却下。

## 5. トレードオフ

- コンセプトを主役（大きめ）にしたことで、「何のアプリか（説明）」は二番手の小さな扱いになる。初見の機能理解より、ブランドの世界観を優先する判断。説明文を残すことで機能理解は担保する。
- OGP と同一文言にすることで、将来コンセプトを変えるときは2箇所（OGP とログイン）を同時に直す必要が生じる。一貫性の代償として受け入れる（むしろ「揃えるべきもの」として運用する）。

## 6. 受け入れ条件（Acceptance Criteria）

- [ ] ログイン画面に、ロゴ → N O S H I → 「大切な人との縁を、長く美しく。」→ 区切り線 → 「贈りものの記録と、お返し選び」が縦に並ぶ。
- [ ] 旧文言「贈答を、ちゃんと続けられる。」がコードベースから消えている。
- [ ] コンセプト文が `index.html` の meta description / og:description と一字一句一致している。
- [ ] 説明文が `<title>` の「贈りものの記録と、お返し選び」部分と一致している。
- [ ] 新規 CSS は既存トークンのみ使用（新しい色・影・グラデの追加がない）。
- [ ] `biome` / `tsc` / `vite build` が通る。

## 7. エッジケース・横断的考慮

- **狭い画面での折り返し**：コンセプト文（13文字）は最小幅（375px）でも1行に収まる想定。説明文（13文字）も同様。区切り線は固定幅 38px で折り返しの影響を受けない。実機で 375px 幅を確認すること。
- **文字を大きくするトグル**（マイページの font-scale ~1.12–1.18×）：相対単位（トークンの px ベースだが rem 換算）でスケールに追従するか確認。少なくとも破綻（重なり・はみ出し）しないこと。
- **弔事（mourning）テーマ**：ログイン画面は `.mourning` スコープの対象外（ログイン前なので世帯のモードは未確定）。影響なし。
- **アクセシビリティ**：区切り線は装飾なので `aria-hidden="true"`。コンセプト・説明は通常のテキストとして読み上げ対象。

## 8. 検証

- 見た目の変更のため `biome` / `tsc` / `vite build` で機械的に担保。
- 文言の自動テストは追加しない（過剰。受け入れ条件の目視確認で足りる）。
- 本番デプロイ後、実ブラウザでヘッダの3層表示と 375px 幅での折り返しを確認。
