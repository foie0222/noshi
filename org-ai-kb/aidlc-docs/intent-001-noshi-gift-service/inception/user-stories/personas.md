# Personas — noshi（MVP）

## P-1 個人の贈答管理者（主ペルソナ）

- **名前/像:** 田中 美和（38歳）。共働き・子育て世代。結婚・出産・入学・お中元お歳暮・香典など、年に何度も贈答イベントに直面する。
- **役割:** 自分（と世帯）の「もらった/あげた」を記録し、適切なお返しを期限内に行う当事者。
- **Goals:**
  - もらった贈答を取りこぼさず記録し、誰に何をいくら頂いたかを後から正確に思い出したい。
  - 失礼にならない金額・品物・礼状で、マナーに沿ったお返しをしたい。
  - お返しのやり残し（出し忘れ）をなくしたい。
- **Context:** 多忙でまとまった時間が取れない。スマホ中心。冠婚葬祭のマナーに自信がなく、その都度ネット検索している。贈答相手の個人情報（氏名・続柄・金額）を扱う後ろめたさ・漏洩への不安がある。

## P-2 attacker（abuse case 用・人間ではない利用想定の脅威主体）

- **像:** 他人の noshi アカウントや保存データ（贈答相手の氏名・金額・関係性＝機微 PII）を不正に取得・閲覧しようとする第三者。アカウント乗っ取り、他ユーザーIDのデータ参照、総当たりログイン、不正な画像/プロンプト投入などを試みる。
- **役割:** 正規ユーザーではない。abuse / security ストーリーの発生源として、システムが「拒否すべき振る舞い」を定義するために置く。
- **Goals（＝防ぐべき脅威）:** 他者データの閲覧・窃取、認証回避、サービス悪用。
- **Context:** noshi は本人が「第三者（贈り先）の PII」を登録する構造のため、漏洩時の影響範囲が本人に閉じない。OWASP A01（アクセス制御）・A07（認証）が特に関連。
