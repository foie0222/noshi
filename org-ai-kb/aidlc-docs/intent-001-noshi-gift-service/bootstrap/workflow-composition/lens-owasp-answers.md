# OWASP Lens — One-time Tailoring Answers

構想初期のための暫定値。requirements-analysis ステージで確定・精緻化する。

- **扱うデータの機微度:** confidential 中心（氏名・住所・贈り先などの PII）＋ 決済を内製する場合は restricted。
- **コンプライアンス:** 暫定で個人情報保護法（APPI）相当。決済導入時に PCI-DSS を再評価。
- **認証モデル:** 暫定でセッション/OAuth2（外部 IdP 利用を想定）。requirements で確定。
- **公開範囲:** インターネット公開（消費者向け Web サービス）。
- **想定脅威:** 公開消費者向けゆえ A01（アクセス制御不備・他人の注文/贈答先の閲覧）、A03（注文フォームの injection）、A07（認証不備）が特に関連。
- **リスク許容度:** バランス型。個人情報漏洩は重大視するが、構想段階では過剰な統制を避ける。
